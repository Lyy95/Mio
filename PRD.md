# 产品需求文档 (PRD): EmotiCare - 智能身心状态伴侣

> **版本**: V1.1  
> **状态**: 草稿  
> **日期**: 2026-02-01  
> **更新日志**: 新增睡眠数据自动同步功能

## 1. 项目背景与目标 (Background)
现代人生活节奏快，普遍面临工作压力、睡眠障碍和情绪波动问题。市面上的记录工具（如待办事项、睡眠追踪器）往往冰冷且割裂，缺乏对用户“整体状态”的关怀。

**EmotiCare** 旨在成为一款**有温度的智能身心伴侣**。它不只是记录数据的工具，而是通过对话式交互（Agent）和自动化的数据采集，全方位感知用户的身心状态，提供即时的情绪价值反馈和深度的健康评估。

**核心理念**：
*   **From Data to Care**: 从单纯的数据记录转向情感关怀。
*   **Conversational First**: 摒弃枯燥的表单，通过自然对话完成记录。
*   **Seamless Integration**: 自动化采集生理数据，减少用户输入负担。

## 2. 用户画像 (User Personas)

### 2.1 焦虑的职场人 (Alex)
*   **特征**：28岁，互联网大厂PM，经常加班，佩戴 Apple Watch。
*   **痛点**：感觉每天都很累，但不知道累在哪里；情绪时常失控；忘记照顾自己。
*   **需求**：需要有人（或AI）在他深夜下班时说一句“辛苦了”，并自动分析昨晚睡眠质量，提醒他该休息了。

### 2.2 独居的自我提升者 (Bella)
*   **特征**：24岁，自由职业者，独居。
*   **痛点**：缺乏社交，情绪无人倾诉；希望通过量化数据来优化自己的工作效率和生活质量。
*   **需求**：需要一个能随时聊天的树洞；需要客观的数据分析来验证自己的生活方式是否健康。

| 角色 | 行为 (Action) | 目的/价值 (Benefit) |

## 4. 功能需求 (Functional Requirements)

### 4.1 核心功能：智能体伴侣 (The Agent Interface)
*   **对话式记录**：
    *   用户通过聊天界面输入内容（支持文本/语音）。
    *   AI 自动提取关键指标：`Mood` (1-10), `Work_Load`, `Energy_Level`。
    *   **情感反馈**：AI 必须对用户的每一条输入给予共情回应。

### 4.2 健康数据自动同步 (Health Data Auto-Sync)
*   **第三方平台接入**：
    *   **Apple HealthKit (iOS)**: 自动获取睡眠时长、入睡时间、醒来时间、心率变异性 (HRV)。
    *   **Google Health Connect (Android)**: 对接安卓生态的睡眠数据。
*   **数据融合**：
    *   当检测到昨晚睡眠不足（如 < 6小时）时，AI 会在早安问候中主动提及：“昨晚似乎睡得很少，今天要注意休息哦”。
    *   若用户未佩戴设备，仍支持通过对话手动补充睡眠数据。

### 4.3 状态分析引擎 (Status Analysis Engine)
*   **多维评分模型**：结合心情、**客观睡眠数据**、工作时长计算每日 `Wellness Score` (身心指数)。
*   **异常检测**：
    *   **规则引擎**：若“心情 < 3”持续 3 天，或“睡眠 < 5小时”持续 3 天。
    *   **触发动作**：推送紧急关怀通知，建议休息或寻求帮助。

### 4.4 月度评估报告 (Monthly Insight Report)
*   **数据可视化**：
    *   心情曲线图 (Mood Timeline)。
    *   **睡眠质量趋势图** (Sleep Quality Trend)。
    *   睡眠与工作压力的相关性分析 (Correlation Chart)。
*   **AI 总结**：
    *   生成一段像心理咨询师口吻的月度评语。
    *   提取“本月关键词”（如：焦虑、突破、疲惫）。
    *   提供下个月的建议（如：尝试增加 10% 的户外活动）。

### 4.5 个人中心与设置
*   **隐私模式**：支持本地数据存储或加密上传。
*   **数据授权管理**：用户可随时开启或关闭健康数据读取权限。
*   **个性化配置**：设置 AI 的性格（温柔型、理智型、幽默型）。

## 5. 非功能需求 (Non-Functional Requirements)
*   **隐私安全**：用户的对话数据极其敏感，必须严格加密；**健康数据仅用于分析，绝不违规出售**。
*   **响应速度**：对话回复延迟应 < 2秒。
*   **可用性**：界面设计应温馨、治愈。

## 6. 数据埋点与指标 (Success Metrics)
*   **DAU/MAU**：用户粘性。
*   **健康数据授权率**：授权自动同步睡眠数据的用户比例。
*   **记录完成率**：用户每天完成至少一次对话记录的比例。
*   **情感互动深度**：用户平均每次对话的轮数。

## 7. 架构简述 (Technical Preview)
*   **前端**：React Native / Flutter (跨平台移动端，方便调用原生健康 API)。
*   **后端**：Node.js/Python。
*   **AI 模型**：LLM API (意图识别 + 情感分析)。
*   **数据库**：PostgreSQL (指标) + MongoDB (日志)。
